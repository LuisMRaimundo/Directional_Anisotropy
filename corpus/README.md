# Corpus and benchmark package (Directional_Anisotropy)

**Directional_Anisotropy** is a **systematic notational directional-field analyzer**: it measures directionality in symbolic **pitch–time transitions** from MusicXML. It does **not** measure audio, perception, harmony, or general texture.

## Status (this release)

| Item | Status |
|------|--------|
| Official representative benchmark corpus | **Not established** |
| Registered entries | 3 **synthetic fixtures** (see `manifest.json`) |
| Frozen reference outputs | Pipeline outputs for available synthetic fixtures |
| Reproducible tables | `corpus/tables/` (generated by scripts) |

Synthetic fixtures support **behavioural regression** and smoke tests. They must **not** be described as a representative repertoire corpus.

## Layout

```
corpus/
  manifest.json           # required metadata per candidate file
  fixtures/               # optional copies (canonical paths may point to tests/fixtures/)
  reference_outputs/      # frozen JSON per corpus_id + config profile
  reference_outputs/checksums.sha256
  tables/                 # reproduced summary tables
  scripts/
    generate_reference_outputs.py
    compare_reference_outputs.py
    reproduce_tables.py
```

## Adding a real benchmark (future)

1. Obtain MusicXML with **verified license** (`public_domain_verified`, `openly_licensed`, or `owned_by_author`).
2. Add a manifest entry with complete metadata.
3. Set `include_in_official_benchmark: true` only when license and representativeness are documented.
4. Run `python corpus/scripts/generate_reference_outputs.py` and commit frozen outputs + checksums.
5. Update `CORPUS_REFERENCIA.md` and `docs/anisotropia_current_rating.md`.

Do **not** import copyrighted scores or invent composer/work metadata.

## Standard analysis profile (frozen outputs)

Defined in `corpus/scripts/generate_reference_outputs.py` as `BENCHMARK_CONFIG`:

- `window_mode`: `total`
- `bootstrap_ci`: `false` (deterministic frozen metrics)
- Default voice-aware horizontal ontology (`legacy_mixed_mode`: false)
